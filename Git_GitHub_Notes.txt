Clone & Status
1. Clone - cloning a repository on our local machine
	git clone <-repository link ->

2. Status: command displays the state of your working directory and staging area, showing you which files have changed and are ready to be committed.
	git status

Steps of Status in Git:

Modify/New -> add -> commit

On branch [branch-name] 	: The name of the branch you are currently on.
Untracked files			: Files that are created new and not yet managed by Git.
Modified			: Files that are changed
Changes to be committed		: Files that you have added to the staging area with git add, which are ready for a commit.
Changes not staged for commit	: Files that have been modified but are not yet added to the staging area.

Add & Commit
1. Add: Adds new or changed files in your working directory to the Git staging area.
	git add <-file name-> 	for single file 
	git add .    		for all files which is modify.

2. Commit: it is the record of change
	git commit -m "Some message"

Push Command
1. Push: upload local repo content to remote repo
	git push origin main

Init Command
1. Init: This creates a hidden .git folder ‚Äî meaning your project is now a local Git repo.
	git init

2. git remote add origin: It is used to establish a connection between an existing local Git repository and a remote repository. This is typically used when you have a local project and want to link it to a newly created or existing remote repository.

	git remote add origin <repo link>



Git Branches

1.	git branch		(to check out the branch name)

2.	git branch -M main	(to rename the branch name)

3.	git checkout <branch name)	(to change branch from current to new)

4. 	git checkout -b <new branch name>	(to create new branch)

5. 	git branch -d <branch name>		(to delete branch)



Merging Code

1. git diff <branch name>		(to compare commits, branches, files & more between present branch and <branch name>)

2. git merge <branch name>		(to merge 2 branches between present branch and <branch name>)


Create PR (Pull Request): It lets you tell others about changes you've pushed to a branch in a repository on GitHub.


Pull Command: Used to fetch and download content from a remote repo and immediately update the local repo to match that content

	git pull origin main


Merge Conflicts: An event that takes place when Git is unable to Automatically resolve differences in code between two commits.

	git merge <branch-name>


Undoing Changes: 

Case 1: staged Changes
	git reset <file name>
	git reset		(all files)

Case 2: committed Changes (for one commit)
	git reset HEAD~1

Case 3: committed changes (for many commits)
	git reset <commit has>
	git reset --hard <commit hash>
	git log 		(to check history of commits)


Fork: Forks are commonly used in open-source development. If you want to propose changes to a project you don't own, you fork it, make your modifications in your fork, and then submit a "pull request" to the original repository's maintainers.

Got it üëç Let me give you a basic Git workflow (flow) that most developers follow. Think of it like the everyday cycle when working with code:


üîÑ Basic Git Flow

1. Initialize / Clone a Repo

   If starting a new project:
   Command:
    	git init
     
   If joining an existing project:
   Command:
   	git clone <repo-url>
     

2. Check Status

   See which files are changed or untracked:
   Command:
   	git status
     
3. Make Changes
    
   Edit, add, or delete files in your project.

4. Stage Changes

   Add specific files:
   Command:
   	git add filename
     
   Or add all changes:
   Command:
   	git add .
     

5. Commit Changes

   Save changes with a message:
   Command:
   	git commit -m "Meaningful message about what you did"
     

6. Push to Remote

   Send commits to GitHub/GitLab/Bitbucket:
   Command:
   	git push origin branch-name
     

7. Pull Updates

   Get latest changes from remote before working again:
   Command:
    	git pull origin branch-name
     

8. Branching (Optional but Common)

   Create a new branch for a feature:
   Command:
   	git checkout -b feature-branch
     
   Switch back to main branch:
   Command:
   	git checkout main
     
   Merge branch when done:
   Command:
   	git merge feature-branch
